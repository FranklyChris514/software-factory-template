# Release / Distribution Checklist

Work through this before any public or distribution step (posting publicly, sharing a link widely, submitting somewhere). This is the one checklist that involves the outside world, so it's also the one that most needs an owner's explicit go-ahead.

- [ ] The product itself has passed the verification checklist.
- [ ] The product is deployed and the deployment checklist is complete.
- [ ] Identify the specific channel (community, platform, person) and confirm its rules allow this kind of post.
- [ ] Draft the actual post/message content, not just a plan to write one.
- [ ] Confirm which account will be used to post, and that it belongs to the project owner or has their explicit sign-off.
- [ ] Get explicit owner approval for: which channel, which account, and the exact content, before anything goes out.
- [ ] After posting, note where and when in the Decisions log or operating doc so it isn't duplicated later.
- [ ] Set up a way to capture what happens next (replies, clicks, sign-ups) so it feeds into a Reality Report.
