# Verification Checklist

Work through this before calling anything "done." Verifying means checking with real evidence, not assuming.

- [ ] Open the actual live/running product yourself and use it end-to-end as a real user would.
- [ ] Check that every claim in the README/description matches what the product actually does right now.
- [ ] Test the core flow with at least one unexpected or edge-case input (empty field, huge number, special characters).
- [ ] If there's a deployment step, confirm the deployed version matches the latest commit (not a stale cache).
- [ ] Check for broken links, broken images, and console errors on the live page.
- [ ] If claiming something works on mobile, actually check it at a small viewport width.
- [ ] Write down what you verified and how, so "done" is backed by evidence, not confidence.
